# Stakeholder Analysis



| Stakeholder          | Role                                       | Key Concerns                    | Pain Points                      | Success Metrics                        |
| -------------------- | ------------------------------------------ | ------------------------------- | -------------------------------- | -------------------------------------- |
| General User         | Uses the system to verify news credibility | Accurate results, fast response | Difficulty identifying fake news | 90% accuracy in detection, results within 2 seconds |
| Journalist           | Verifies sources before publishing         | Reliable verification tools     | Time-consuming fact-checking     |Reduced verification time by 50%                |
| Researcher           | Studies misinformation trends              | Access to analyzed data         | Lack of structured datasets      |Easy export of analyzed data                      |
| System Administrator | Maintains system infrastructure            | System uptime and reliability   | System crashes or downtime                  | 99% system uptime                            |
| Developer            | Builds and maintains the system            | Maintainable and scalable code  | Poor documentation               | Easy updates and modular system design                         |
| Organization         | Uses system for awareness                  | Trust and impact                | Spread of misinformation         |Increased public awareness by usage metrics               |
