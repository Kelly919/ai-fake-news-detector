# Stakeholder Analysis

## 1. General User

**Role:** Uses the system to verify news credibility

**Key Concerns:** Accurate results, fast response

**Pain Points:** Difficulty identifying fake news

**Success Metrics:** 90% accuracy in detection, results within 2 seconds

---

## 2. Journalist

**Role:** Verifies sources before publishing

**Key Concerns:** Reliable verification tools

**Pain Points:** Time-consuming fact-checking

**Success Metrics:** Reduced verification time by 50%

---

## 3. Researcher

**Role:** Studies misinformation trends

**Key Concerns:** Access to analyzed data

**Pain Points:** Lack of structured datasets

**Success Metrics:** Easy export of analyzed data

---

## 4. System Administrator

**Role:** Maintains system infrastructure

**Key Concerns:** System uptime and reliability

**Pain Points:** System crashes or downtime

**Success Metrics:** 99% system uptime

---

## 5. Developer

**Role:** Builds and maintains the system

**Key Concerns:** Maintainable and scalable code

**Pain Points:** Poor documentation

**Success Metrics:** Easy updates and modular system design

---

## 6. Organization / Institution

**Role:** Uses system for public awareness

**Key Concerns:** Trustworthiness and impact

**Pain Points:** Spread of misinformation

**Success Metrics:** Increased public awareness by usage metrics
